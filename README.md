## 📌 Project Overview

The objective of this project is to design and implement a scalable healthcare data platform capable of ingesting, transforming, and analyzing hospital data to generate meaningful business insights.

This project demonstrates an end-to-end Azure Data Engineering solution built using a healthcare dataset from Cleveland Clinic. The project focuses on ingesting raw CSV files, performing data cleaning and transformation using Azure Data Factory Mapping Data Flows, storing cleaned data in Azure SQL Database, and generating analytical reports using SQL and Python.

## 🏗️ Project Workflow

### 1. Data Ingestion

- Raw CSV files are stored in ADLS Gen2.
- Azure Data Factory pipelines ingest the files into Azure SQL raw tables.

### 2. Data Cleaning and Transformation

Mapping Data Flows were used to:

- Remove leading and trailing spaces.
- Convert dates into standard format.
- Convert Yes/No values into Boolean fields.
- Correct data types.
- Handle duplicates and inconsistencies.

### 3. Master Pipeline Execution

A master pipeline orchestrates the execution of all data cleaning pipelines and ensures dependency management between datasets.

### 4. Analytics Layer

Azure Synapse and SQL queries were used to generate insights such as:

- Branch revenue analysis
- Most common diagnoses
- Doctor prescription analysis
- Patient visit trends

### 5. Automated Reporting

Python scripts automatically generate Excel reports from Azure SQL Database.
